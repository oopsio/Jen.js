Islands Architecture

Islands are interactive components that are hydrated client-side while the rest of your page remains static HTML.

What is an Island?

An island is a component that needs JavaScript to function. Everything else is static HTML.

import { Island } from "@jenjs/master/src/runtime/islands.js";
import { useState } from "preact/hooks";

function CounterImpl({ initial }: { initial: number }) {
  const [count, setCount] = useState(initial);
  return (
    <button onClick={() => setCount(c => c + 1)}>
      Count: {count}
    </button>
  );
}

const Counter = Island(CounterImpl, "load");

export default function Page() {
  return (
    <main>
      <h1>My Page</h1>
      <p>This is static HTML - no JavaScript.</p>
      <Counter initial={0} /> {/* This is hydrated */}
      <p>This is also static HTML.</p>
    </main>
  );
}

Hydration Strategies

Control when your islands load JavaScript:

load - Immediate

Hydrate as soon as the page loads.

const Counter = Island(CounterImpl, "load");

Use for:

  • Critical interactive components
  • Above-the-fold content
  • Navigation elements

idle - When Browser is Idle

Hydrate when the browser has free CPU time using requestIdleCallback.

const Counter = Island(CounterImpl, "idle");

Use for:

  • Secondary interactive components
  • Non-critical features
  • Below-the-fold content

visible - When in Viewport

Hydrate only when the element scrolls into view using IntersectionObserver.

const Counter = Island(CounterImpl, "visible");

Use for:

  • Far-down-the-page elements
  • Modal dialogs
  • Lazy-loaded content

Performance Benefits

By using islands strategically, you get:

  • Smaller bundles - Only interactive code ships
  • Faster loads - Less JavaScript to parse and execute
  • Better performance - Idle/visible strategies defer hydration
  • Progressive enhancement - Content available before JS loads

Example: Blog Post

import { Island } from "@jenjs/master/src/runtime/islands.js";
import Comments from "../components/Comments.tsx";
import Newsletter from "../components/Newsletter.tsx";

// Comments need JS for form handling
const CommentsIsland = Island(Comments, "visible");

// Newsletter form for below the fold
const NewsletterIsland = Island(Newsletter, "idle");

export default function BlogPost({ data }: { data: { post: Post } }) {
  return (
    <main>
      <article>
        <h1>{data.post.title}</h1>
        <p>{data.post.content}</p> {/* Static HTML */}
      </article>

      {/* Hydrate when scrolled into view */}
      <CommentsIsland postId={data.post.id} />

      {/* Hydrate when browser is idle */}
      <NewsletterIsland />
    </main>
  );
}

Bundle Size Impact

Without islands (traditional SPA):

Total JS: 250KB
  - Framework: 50KB
  - Hydration code: 80KB
  - App code: 120KB

With islands:

Total JS: 85KB
  - Framework: 30KB
  - Island code: 55KB (only used components)

Result: 66% smaller JavaScript bundle!

Passing Props

Islands accept props like normal components:

const Button = Island(ButtonImpl, "load");

export default function Page() {
  return (
    <Button 
      label="Click me" 
      onClick={() => alert('Clicked!')} 
    />
  );
}

Props are serialized to JSON, so they must be serializable:

// ✅ Works
<Counter initial={5} />
<Button label="Click" />

// ❌ Doesn't work (functions not serializable)
<Button onClick={() => {}} />
<Counter onTick={() => {}} />

Advanced Example: Shopping Cart

import { Island } from "@jenjs/master/src/runtime/islands.js";
import ShoppingCart from "../components/ShoppingCart.tsx";

const CartIsland = Island(ShoppingCart, "visible");

export default function ProductPage({ data }: { data: { product: Product } }) {
  return (
    <main>
      <section>
        <h1>{data.product.name}</h1>
        <p>{data.product.description}</p> {/* Static */}
        <img src={data.product.image} alt={data.product.name} /> {/* Static */}
      </section>

      <section>
        Price: ${data.product.price} {/* Static */}
      </section>

      {/* Interactive cart - hydrated when visible */}
      <CartIsland productId={data.product.id} price={data.product.price} />

      <section>
        <h2>Reviews</h2>
        {data.product.reviews.map(review => (
          <div key={review.id}>{review.text}</div> {/* Static */}
        ))}
      </section>
    </main>
  );
}

Best Practices

  1. Use islands sparingly - More islands = more JavaScript
  2. Prefer visible for below-the-fold - Defer non-critical JS
  3. Keep islands simple - Complex logic in parent page
  4. Serialize data carefully - Props must be JSON-serializable
  5. Test without JS - Verify fallbacks work
  6. Monitor bundle size - Track island code impact
