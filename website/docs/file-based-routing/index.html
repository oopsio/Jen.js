File-Based Routing

Jen.js uses an intuitive file-based routing system. Routes are automatically created based on your file structure in the site/ directory.

Basic Routing

Files in site/ become routes based on their parenthetical filenames:

FileRoute
site/(home).tsx/
site/(about).tsx/about
site/(contact).tsx/contact
site/blog/(post).tsx/blog/post
site/api/(users).ts/api/users

Parenthetical Naming

The text inside parentheses becomes the route path:

site/
├── (home).tsx        → /
├── (about).tsx       → /about
├── (pricing).tsx     → /pricing
└── blog/
    ├── (post).tsx    → /blog/post
    └── (archive).tsx → /blog/archive

Dynamic Routes

Create dynamic segments with bracket notation:

site/
├── (home).tsx
└── blog/
    └── ([id]).tsx    → /blog/:id

Access route parameters in your loader:

export async function loader(ctx: LoaderContext) {
  const { id } = ctx.params;
  const post = await getPost(id);
  return { post };
}

export default function BlogPost({ data }: { data: { post: Post } }) {
  return <h1>{data.post.title}</h1>;
}

Catch-All Routes

Use [...slug] to catch all remaining segments:

site/
└── docs/
    └── ([...slug]).tsx  → /docs/* → /docs/a/b/c

Access the caught segments:

export async function loader(ctx: LoaderContext) {
  const { slug } = ctx.params;
  // slug = ['guides', 'getting-started']
  return { slug };
}

Nested Layouts

Create shared layouts for nested routes:

site/
├── (home).tsx
├── blog/
│   ├── _layout.tsx    // Shared layout
│   ├── (index).tsx    → /blog
│   ├── ([id]).tsx     → /blog/:id
│   └── (archive).tsx  → /blog/archive

The _layout.tsx wraps all nested routes:

export default function BlogLayout({ children }) {
  return (
    <div class="blog">
      <header>Blog</header>
      {children}
      <footer>Blog Footer</footer>
    </div>
  );
}

Multiple Segments

Create deeply nested routes naturally:

site/
├── (home).tsx
└── docs/
    ├── (intro).tsx           → /docs/intro
    └── guides/
        ├── (setup).tsx       → /docs/guides/setup
        └── ([category])
            └── ([id]).tsx    → /docs/guides/:category/:id

Optional Routes

Use question marks for optional segments:

site/
└── blog/
    └── ([id?]).tsx  → /blog, /blog/123

Route Patterns

The default route pattern is /^\((.+)\)\.(t|j)sx?$/ which matches:

  • (pageName).tsx - TypeScript React
  • (pageName).jsx - JavaScript React
  • (pageName).ts - TypeScript API route
  • (pageName).js - JavaScript API route

Important Notes

  • ✅ Routes are matched by filename, not folder name
  • ✅ Index files default to the directory path
  • ✅ Dynamic segments use bracket notation: [id]
  • ✅ Catch-all routes: [...slug]
  • ✅ Parentheses are required around the page name

Blog with Archives

│   │   ├── (setup).tsx   → /docs/guides/setup
│   │   └── ([...path]).tsx → /docs/guides/*
